LMS Stripe Payment Integration Plan

1) What we built so far

Backend flow:
- Added a Payment model to store Stripe payment records.
- Added a Stripe helper utility.
- Added payment controller endpoints for payment intent creation, confirmation, payment status, and webhook handling.
- Added payment routes and mounted them in the Express app.
- Linked payment success to enrollment creation.
- Blocked manual enrollment for paid courses so checkout is required.

Frontend flow:
- Installed Stripe client packages.
- Added API service methods for payment creation, confirmation, and status.
- Added a dedicated enroll checkout page.
- Added a Stripe checkout form component.
- Updated course detail call-to-action links to point to the checkout route.

4) Backend implementation steps

Step 1:
- Create a Payment model that stores student, course, amount, currency, status, Stripe PaymentIntent ID, receipt URL, and metadata.

Step 2:
- Create a Stripe helper that exports the Stripe SDK instance using STRIPE_SECRET_KEY.

Step 3:
- Add payment controller actions:
  - createPaymentIntent
  - confirmPayment
  - getPaymentStatus
  - handleStripeWebhook

Step 4:
- Add payment routes:
  - POST /api/payments/create-intent
  - POST /api/payments/confirm
  - GET /api/payments/get-status/:paymentIntentId

Step 5:
- Mount the webhook endpoint in app.js using raw body parsing before express.json.

Step 6:
- On successful payment, create or attach the enrollment record and increment course enrolledCount.

Step 7:
- Prevent direct enrollment for paid courses so checkout is the only path for payment-based access.
