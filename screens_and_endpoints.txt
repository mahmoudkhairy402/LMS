### General (Public/Unauthenticated Users)

---

#### 1. Homepage / Public Course Catalog
*   **User Role**: Anyone (Unauthenticated)
*   **Description**: Displays a list of all published courses that anyone can browse.
*   **Endpoint**: `GET /api/courses`
    *   **Request Body**: None.
    *   **Query Params (for filtering/pagination)**:
        *   `page`: (number) The page number to retrieve (e.g., `1`).
        *   `limit`: (number) The number of courses per page (e.g., `10`).
        *   `category`: (string) Filter courses by a specific category.
        *   `level`: (string) Filter courses by difficulty level (e.g., `Beginner`).
        *   `instructor`: (string) Filter courses by instructor ID.
        *   `search`: (string) Search by course title, description, or tags.
    *   **Response**: A paginated list of published courses.
        ```json
        {
          "success": true,
          "page": 1,
          "limit": 10,
          "total": 50, // Total number of courses matching the filter
          "totalPages": 5,
          "courses": [
            {
              "title": "Web Development Basics",
              "slug": "web-development-basics",
              "shortDescription": "An introduction to web development.",
              "thumbnail": "url/to/image.png",
              "category": "Web Development",
              "instructor": { "name": "John Doe", ... },
              // ... other course metadata
            }
          ]
        }
        ```
    *   **Pagination**: Yes. Uses `page` and `limit` query parameters. The response includes `total`, `totalPages`, `page`, and `limit`.

### Student Role

---

#### 1. Register Screen
*   **User Role**: Student (or any new user)
*   **Description**: A form for new users to create an account.
*   **Endpoint**: `POST /api/auth/register`
    *   **Request Body**:
        ```json
        {
          "name": "Sarah Chen",
          "email": "sarah.chen@gmail.com",
          "password": "securepassword123",
          "role": "student" // Can be 'student' or 'instructor'
        }
        ```
    *   **Response**: A success message indicating that a verification email has been sent.
        ```json
        {
          "success": true,
          "message": "User registered successfully. Please check your email to verify your account."
        }
        ```
    *   **Pagination**: No.

---

#### 2. Login Screen
*   **User Role**: Student, Instructor, Admin
*   **Description**: A form for existing users to log in.
*   **Endpoint**: `POST /api/auth/login`
    *   **Request Body**:
        ```json
        {
          "email": "sarah.chen@gmail.com",
          "password": "securepassword123"
        }
        ```
    *   **Response**: The user's profile information and an access token.
        ```json
        {
          "success": true,
          "message": "Logged in successfully",
          "user": {
            "id": "user_id",
            "name": "Sarah Chen",
            "email": "sarah.chen@gmail.com",
            "role": "student"
          },
          "accessToken": "jwt.access.token"
        }
        ```
    *   **Pagination**: No.

---

#### 3. Course Details Screen
*   **User Role**: Student (and Public)
*   **Description**: Shows detailed information about a single course, including its description, instructor, and lessons (previews for unenrolled users).
*   **Endpoint**: `GET /api/courses/:id`
    *   **Request Body**: None.
    *   **Response**: The full details of the specified course.
        ```json
        {
          "success": true,
          "course": {
            "_id": "course_id",
            "title": "Web Development Basics",
            "instructor": { "name": "John Doe", ... },
            "sections": [
              {
                "title": "Introduction",
                "lessons": [ { "title": "Welcome!", "isPreview": true } ]
              }
            ]
            // ... other course details
          }
        }
        ```
    *   **Pagination**: No.

        ```json
              // ... other enrollment details
            }
          ]
        }
        ```
    *   **Pagination**: No.

#### 5. Course Player Screen
*   **User Role**: Student
*   **Description**: The main learning interface where a student watches lessons, reads articles, and updates their progress.
*   **Endpoint**: `PUT /api/enrollments/my/:id/progress`
    *   **Description**: This endpoint is called when a student marks a lesson as complete or their progress is otherwise updated.
    *   **Request Body**:
        ```json
        {
          "progressPercent": 30,
          "completedLessonIds": ["lesson_id_1", "lesson_id_2"]
        }
        ```
    *   **Response**: The updated enrollment object.
        ```json
        {
          "success": true,
          "message": "Progress updated successfully",
          "enrollment": {
            // ... updated enrollment details
          }
        }
        ```
    *   **Pagination**: No.

#### 4. All My Students Screen
*   **User Role**: Instructor
*   **Description**: Shows all students enrolled across all of the instructor's courses.

        *   `courseId`: (string) Filter students by a specific course ID.
        *   `enrollmentStatus`: (string) Filter by status (e.g., `active`, `completed`).
        *   `search`: (string) Search by student name or email.
    *   **Response**: A paginated list of students.
        ```json
        {
          "success": true,
          "page": 1,
          "limit": 10,
          "total": 500,
          "totalPages": 50,
          "students": [
            {
              "studentId": "student_id_1",
              "name": "Sarah Chen",
              "email": "sarah.chen@gmail.com",
              "progressPercent": 100,
              "courseName": "Advanced React Patterns"
            }
          ]
        }
        ```
    *   **Pagination**: Yes. Uses `page` and `limit`.

### Admin Role

---

#### 1. User Management Dashboard
*   **User Role**: Admin
*   **Description**: A table displaying all users on the platform, with tools for filtering, sorting, and performing bulk actions.
*   **Endpoint**: `GET /api/users`
    *   **Request Body**: None.
    *   **Query Params**:
        *   `page`, `limit`: For pagination.
        *   `role`: (string) Filter by `student`, `instructor`, or `admin`.
        *   `status`: (string) Filter by `active` or `inactive`.
        *   `isEmailVerified`: (string) `true` or `false`.
        *   `search`: (string) Search by user name or email.
        *   `sortBy`: (string) Field to sort by (e.g., `createdAt`, `name`).
        *   `sortOrder`: (string) `asc` or `desc`.
    *   **Response**: A paginated list of all users.
        ```json
        {
          "success": true,
          "page": 1,
          "limit": 10,
          "total": 1200,
          "totalPages": 120,
          "users": [
            {
              "_id": "user_id",
              "name": "Test User",
              "email": "test@example.com",
              "role": "student",
              "isActive": true,
              "enrollmentCount": 5,
              "courseCount": 0
            }
          ]
        }
        ```
    *   **Pagination**: Yes. Uses `page` and `limit`.

---

#### 2. User Detail/Edit Screen
*   **User Role**: Admin
*   **Description**: View and edit the details of a single user.
*   **Endpoints**:
    *   `GET /api/users/:id` (to fetch user data)
    *   `PUT /api/users/:id` (to update user data)
*   **Request Body** (`PUT`):
    ```json
    {
      "name": "Updated Name",
      "role": "instructor",
      "isActive": false,
      "isEmailVerified": true
    }
    ```
*   **Response**: The user object.
*   **Pagination**: No.

#### 5. Bulk Action Modals
*   **User Role**: Admin
*   **Description**: Modals or forms to perform bulk operations on selected users from the User Management Dashboard.
*   **Endpoints**:
    *   `POST /api/users/bulk/deactivate`
    *   `POST /api/users/bulk/activate`
    *   `POST /api/users/bulk/update-role`
    *   `POST /api/users/bulk/delete`
*   **Request Body** (Example for `update-role`):
    ```json
    {
      "userIds": ["id1", "id2", "id3"],
      "newRole": "instructor"
    }
    ```
*   **Response**: A summary of the operation's success or failure.
    ```json
    {
      "success": true,
      "message": "3 users updated successfully",
      "results": { "successCount": 3, "failureCount": 0 }
    }
    ```
*   **Pagination**: No.
